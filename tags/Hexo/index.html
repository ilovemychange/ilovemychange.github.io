Hexo와 GitHub로 기술 블로그 서비스하기

먼저 Hexo란, 빠르고 간단하고 파워풀한 블로그 프레임워크이다. 필자는 Bear라는 메모앱으로 Markdown 노트들을 기록하고 있는데, Hexo 역시 Markdown 언어를 활용하여 포스트를 작성할 수가 있어서 둘을 호환할 수 있겠다는 생각을 했다.

1. 설치하기

기본 세팅 - node.js 와 Hexo 설치

먼저, 기본 세팅에 대해 알아본다. Hexo를 활용하기 위해서는 Node.js, Git 등이 먼저 설치되어 있어야 한다. 관련하여 Hexo Setup 가이드 문서를 참고한다.

Hexo 기본 세팅 및 서버 실행

Hexo 블로그 Github를 이용해 만들어보기 - Mishka’s Blog

GitHub 레포지토리 생성

필자는 Github 블로그 만들기 with Hexo - 배포 및 기타 설정 - 밍피디의 개발노트 와
github 개인 페이지 만들기!를 참고하였다.

2. Theme 적용하기

Hexo의 강력한 기능 중 하나는 바로 테마이다. 전세계 각지의 프론트엔드 전문가들이 Hexo 테마를 제작해두었고, 현재도 제작하고 있다. 테마는 Hexo 공식 테마 리스트에서 확인이 가능하다.

필자는 icarus라는 테마를 적용하였다. 해당 테마 적용에 대해서는 다음 블로그가 큰 도움이 되었다. icarus 테마는 추가적으로 설치해야 하는 플러그인(?)들이 있어서, 일반적인 방식대로만 하면 Error가 발생하니 다음 링크를 참고하길 바란다. 헥소(HEXO)를 이용한 깃허브 블로그 만들기 by icarus 테마

Icarus 테마가 무사히 적용되었다면, _config.icarus.yml 파일을 통해 각종 블로그 기본설정들을 셋팅할 수 있다. 이와 관련해서는 icarus 제작자가 Icarus 가이드 문서를 만들어 놓았으니 참고하길 바란다.

그 외에도 개인 프로필로 활용하기 좋을 것 같은 테마 Resume - Hexo와 고급 개발자 느낌이 물씬 나는 테마 Obsidian - Hexo도 눈여겨보고 있다. Obsidian - Hexo 테마는 다음 블로그를 참고하였다. -> iOS Engineer 정진수

적용 가능한 테마 - alpha-dust, butterfly, next, icarus
실패 이력 - 시도했지만 적용 불가능한 테마들 - Terminal, Obsidian, adoubi, doc, huhu, book, vexo

3. 사용법

사용법은 의외로 간단하다. 필자처럼 노트앱을 통해 Markdown 파일 (.md)를 생성하였다면, 해당 파일을 ::/블로그레포지토리/source/_posts:: 경로 아래에 저장한 후에 바로 빌드/배포를 진행하면 되고, 새롭게 글을 작성하고자 한다면 아래 글쓰기 관련 포스트를 참고한다.
